    # typeof

    typeof用于检测基本类型，当他检测引用类型array和object时，得到的都是object。

    # instanceof

    instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。

    用于检测引用类型，它可以区分出array和object。其内部是通过原型链来实现的，比如f instanceof Foo的判断逻辑是：f的__protp__一层一层往上，能否对应到Foo.prototype。

    判断逻辑翻译成js代码如下

    _instanceof(f, Foo);  

function _instanceof(a, b) {
    while(a){
        if(a.__proto__ === b,prototype ) return true;
        a = a.__proto__;
    }
    return false;
}

    instanceof不准确的原因

    instanceof操作符的问题在于，它假定只有一个全局环境。如果网页中包含多个框架，那实际上就存在两个以上不同的全局执行环境，从而存在两个以上不同版本的Array构造函数。 如果你从一个框架向另一个框架传入一个数组，那么传入的数组与在第二个框架中原生创建的数组分别具有各自不同的构造函数。

    # isArray()

    最终确定某个值到底是不是数组，而不管它是在哪个全局执行环境中创建的，这个方法的用法如下。

    if（Array.isArray(value)）{
    return true;
}

    # Object.prototype.toString

    如果在尚未实现isArray方法的浏览器中准确监测数组,我们需要用到Object.prototype.toString方法来判断，每一个继承自Object的对象都拥有toString的方法。

    //在使用Array的情况下，考虑兼容性问题
if(!Array.isArray){
    Array.isArray = function(arg){
        return Object.prototype.toString.call(arg)==='[object Array]'
    }
}

//自己封装一个获取变量类型的函数
function getType(obj) {
    return Object.prototype.toString.call(obj).slice(8,-1);
}
